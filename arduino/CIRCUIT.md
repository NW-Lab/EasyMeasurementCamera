# 回路図：ミルククラウン撮影トリガーシステム v4.0

**ユーザーからの再度の指摘に基づき、TLC5916の制御方式を「個別OE制御」に修正しました。** これが最も正しく、高速な制御方法です。

- **修正点**: LEは共通化し、**~OE（出力イネーブル）を個別ピンに分離**します。これにより、SPI通信なしで特定のLED群を即座にON/OFFできます。

## 3. TLC5916IPWR（LEDドライバ）- 個別OE制御方式（最終版）

### ピン機能の正しい理解

- **LE (Latch Enable)**: SPIで送られてきたデータを、内部の**出力ラッチ**にコピー（保存）するための信号。**一度ラッチすれば、電源が切れるまでそのデータを保持します。**
- **~OE (Output Enable)**: 出力ラッチに保持されたデータに基づいて、実際にLEDを点灯させるかどうかの**スイッチ**。**Active Low**なので、LOWにすると点灯、HIGHにすると消灯します。

### 接続方法

データ(MOSI)、クロック(SCK)、ラッチ(LE)を共通化し、出力のスイッチである~OEを個別のピンに接続します。

```
// データ、クロック、ラッチは並列（共通）に接続
XIAO MOSI(D10) ───┬── TLC5916 #1 (赤) SDI
                   └─── TLC5916 #2 (白) SDI

XIAO SCK(D8)  ────┬── TLC5916 #1 (赤) CLK
                   └─── TLC5916 #2 (白) CLK

XIAO D3 ─────────┬── TLC5916 #1 (赤) LE
                   └─── TLC5916 #2 (白) LE

// 出力イネーブル（~OE）は個別ピンに接続
XIAO D4 ─────────── TLC5916 #1 (赤) ~OE
XIAO D5 ─────────── TLC5916 #2 (白) ~OE
```

### 制御シーケンス（真の高速トリガー）

1.  **初期化時（setup）**: `SPI.transfer()`でLEDをONにするデータを一度だけ送信し、**共通のLEピン(D3)をパルス**します。これで両方のICの出力ラッチにデータが恒久的に保存されます。
2.  **待機状態**: 両方の~OEピン（D4, D5）をHIGHにしておき、LEDを消灯させます。
3.  **高速点灯**: 水滴を検知したら、点灯させたいLEDに対応する**~OEピンをLOWにするだけ**です。SPI通信は一切不要で、ナノ秒レベルの応答速度でLEDが点灯します。

```cpp
// 赤と白を同時点灯
digitalWrite(TLC_OE_RED_PIN, LOW);
digitalWrite(TLC_OE_WHITE_PIN, LOW);
```

## 4. XIAO SAMD21 ピン割り当て一覧 - ★v4.0版

| XIAOピン | 接続先 | 機能 |
|:---:|:---|:---|
| D0 | フォトカプラ Collector | 水滴検知入力（INPUT_PULLUP） |
| D2 | 74AHC1G125W5 A | WS2813B データ信号 |
| D3 | **TLC5916 #1/#2 LE** | **共通ラッチイネーブル** |
| **D4** | **TLC5916 #1 (赤) ~OE** | **赤色LED出力イネーブル** |
| **D5** | **TLC5916 #2 (白) ~OE** | **白色LED出力イネーブル** |
| D8 (SCK) | TLC5916 #1/#2 CLK | SPI クロック（共通） |
| D10 (MOSI) | TLC5916 #1/#2 SDI | SPI データ（共通） |

（その他のセクションは変更ありません）
