# 回路図：ミルククラウン撮影トリガーシステム v5.0 (最終確定版)

**度重なる修正の末、TLC5916の正しいインターフェース仕様を完全に理解し、最終的な設計を確定しました。**

- **最終結論**: TLC5916はSPIデバイスではなく、**標準的なシフトレジスタ**です。`SPI.transfer()`は使用せず、GPIOを手動で制御（ビットバンギング）する必要があります。
- **設計**: 2つのTLC5916を、それぞれ**4本の独立したGPIOピン**で完全に分離して制御します。これにより、最大限の制御性と柔軟性を確保します。

## 3. TLC5916IPWR（LEDドライバ）- 完全独立制御方式

### 接続方法

2つのTLC5916を、それぞれ4本のGPIOピンで完全に独立して制御します。これにより、相互の干渉が一切なくなり、デバッグも容易になります。

```
// TLC5916 #1 (赤色LED用)
XIAO D6 ─────────── TLC5916 #1 SDI
XIAO D7 ─────────── TLC5916 #1 CLK
XIAO D8 ─────────── TLC5916 #1 LE
XIAO D9 ─────────── TLC5916 #1 ~OE

// TLC5916 #2 (白色LED用)
XIAO D3 ─────────── TLC5916 #2 SDI
XIAO D4 ─────────── TLC5916 #2 CLK
XIAO D5 ─────────── TLC5916 #2 LE
XIAO D10 ────────── TLC5916 #2 ~OE
```

### 制御シーケンス（ビットバンギング）

Arduinoの`SPI`ライブラリは使用しません。`digitalWrite()`と`delayMicroseconds()`を駆使して、データシートのタイミング仕様通りに信号を生成します。

1.  **データ送信 (`shiftOutData`)**: forループを使い、1ビットずつSDIピンの状態をセットし、CLKピンをパルスしてデータをICに送り込みます。
2.  **データラッチ**: データ送信後、LEピンをHIGH→LOWにパルスすると、シフトレジスタの内容が出力ラッチに保存されます。
3.  **高速点灯**: `setup()`でデータをラッチしておけば、トリガー検知時は`digitalWrite(~OE, LOW)`を呼び出すだけで即時点灯します。
