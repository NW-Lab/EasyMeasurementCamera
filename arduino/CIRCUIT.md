# 回路図：ミルククラウン撮影トリガーシステム v3.0

**ユーザーからの提案に基づき、TLC5916の制御を「個別LE制御」方式に変更しました。**

- **修正点**: 2つのTLC5916のLE（ラッチイネーブル）ピンを別々のマイコンピンに接続します。これにより、SPI通信なしで特定のLED群をマイクロ秒単位で高速にON/OFFできるようになり、トリガーの応答性が大幅に向上します。

```
      └── GND      → 全GND共通
```

## 3. TLC5916IPWR（LEDドライバ）- 個別LE制御方式

### 接続方法

カスケード接続は行わず、2つのTLC5916を**独立したデバイス**として扱います。データライン（MOSI）とクロック（SCK）は共通ですが、ラッチ（LE）を個別のピンに接続します。

```
// データとクロックは並列（共通）に接続
XIAO MOSI(D10) ───┬── TLC5916 #1 (赤) SDI
                   └─── TLC5916 #2 (白) SDI

XIAO SCK(D8)  ────┬── TLC5916 #1 (赤) CLK
                   └─── TLC5916 #2 (白) CLK

// ラッチ（LE）は個別ピンに接続
XIAO D3 ─────────── TLC5916 #1 (赤) LE
XIAO D5 ─────────── TLC5916 #2 (白) LE

// 出力イネーブル（~OE）は共通
XIAO D4 ─────────┬── TLC5916 #1 ~OE
                   └─── TLC5916 #2 ~OE
```

### 制御シーケンス（高速トリガー）

1.  **初期化時（setup）**: `SPI.transfer()`でLEDをONにするデータ（`0b00111111`など）を一度だけ送信します。このデータは両方のICのシフトレジスタに同時にロードされます。
2.  **データ保持**: `pulseLE()`で各ICのLEピンを個別にパルスし、シフトレジスタのデータを内部の**出力ラッチ**にそれぞれ保持させます。この時点では`~OE`がHIGHなのでLEDは点灯しません。
3.  **高速点灯**: 水滴を検知したら、`digitalWrite(TLC_OE_PIN, LOW)`を呼び出すだけで、ラッチ済みのデータに基づいて両方のLEDが**即座に（SPI通信なしで）**点灯します。
    - もし個別に点灯させたい場合は、`~OE`をLOWにした状態で、点灯させたい方のLEピン（D3 or D5）をパルスします。

この方式により、トリガーから点灯までの遅延を最小限に抑えることができます。

## 4. XIAO SAMD21 ピン割り当て一覧 - ★v3.0版

| XIAOピン | 接続先 | 機能 |
|:---:|:---|:---|
| D0 | フォトカプラ Collector | 水滴検知入力（INPUT_PULLUP） |
| D2 | 74AHC1G125W5 A | WS2813B データ信号 |
| **D3** | **TLC5916 #1 (赤) LE** | **赤色LEDラッチイネーブル** |
| D4 | TLC5916 #1/#2 ~OE | 出力イネーブル（共通） |
| **D5** | **TLC5916 #2 (白) LE** | **白色LEDラッチイネーブル** |
| D8 (SCK) | TLC5916 #1/#2 CLK | SPI クロック（共通） |
| D10 (MOSI) | TLC5916 #1/#2 SDI | SPI データ（共通） |
| 5V | WS2813B VCC, TLC5916 VLED, 74AHC1G125W5 VCC | 5V電源 |
| 3.3V | TLC5916 VDD | IC電源 |
| GND | 全GND | 共通グランド |

（その他のセクションは変更ありません）
